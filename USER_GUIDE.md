# 그어정 서비스 사용자 안내서

“그거 어디에 정리했더라?”를 해결하는 AI 문서 검색 비서입니다.  
이 서비스는 팀 문서를 빠르게 찾기 위한 사이트이며, 팀원 승인과 문서 권한 구분 기능이 함께 들어 있습니다.

## 1. 이 서비스에서 할 수 있는 일

- 팀 문서를 검색하고 확인할 수 있습니다.
- 문서를 `Public` / `Private`로 나누어 저장할 수 있습니다.
- AI 검색으로 관련 문서를 더 빠르게 찾을 수 있습니다.
- 별도 승인 구역인 `계정 아카이브`에서 로그인 정보, URL, IP 같은 항목을 검색할 수 있습니다.

## 2. 처음 사용할 때

이 사이트는 아무나 바로 들어가는 구조가 아닙니다.  
회원가입을 하고, 이메일 인증을 마친 뒤, 관리자의 승인을 받아야 메인 화면에 들어갈 수 있습니다.

순서는 아래와 같습니다.

1. 회원가입
2. 이메일 인증
3. 관리자 승인
4. 로그인
5. 메인 화면 사용

## 3. 회원가입 방법

로그인 화면에서 아래 정보를 입력합니다.

- 이메일
- 비밀번호

그 다음 `회원가입` 버튼을 누릅니다.

회원가입이 완료되면:

- 인증 메일이 발송됩니다.
- 아직 바로 로그인되는 것은 아닙니다.
- 이메일 인증과 관리자 승인이 모두 끝나야 메인 화면에 들어갈 수 있습니다.

## 4. 이메일 인증

회원가입 후 받은 이메일에서 인증 링크를 눌러야 합니다.

확인할 곳:

- 받은편지함
- 스팸함
- 프로모션함

이메일 인증이 끝나지 않으면 로그인 시 아래 문구가 보일 수 있습니다.

- `이메일 인증 전입니다. 메일함에서 인증을 완료해 주세요.`

## 5. 관리자 승인

이메일 인증이 끝났더라도, 관리자가 승인하기 전까지는 메인 화면에 들어갈 수 없습니다.

승인 상태는 아래 셋 중 하나입니다.

- `승인 대기`
- `승인 완료`
- `승인 거절`

### 승인 대기일 때

- 메인 화면에 들어갈 수 없습니다.
- 승인 대기 화면이 보입니다.

### 승인 거절일 때

- 관리자 승인 재요청을 할 수 있습니다.
- 재요청은 최대 3번까지 가능합니다.

## 6. 로그인

이메일 인증과 관리자 승인이 끝난 뒤 로그인할 수 있습니다.

로그인이 완료되면:

- 문서 검색 화면이 열립니다.
- 본인이 볼 수 있는 문서만 표시됩니다.

## 7. Public / Private 문서 차이

문서는 두 가지 권한으로 나뉩니다.

### Public 문서

- 승인된 팀원이라면 누구나 검색 가능
- 누구나 상세 확인 가능

### Private 문서

- 문서를 만든 본인만 검색 가능
- 문서를 만든 본인만 상세 확인 가능

즉, 팀원 인증을 통과한 사람만 이 사이트에 들어올 수 있고,  
그 안에서 다시 `Public`과 `Private`이 나뉘는 구조입니다.

## 8. 문서 등록과 수정

승인된 사용자는 문서를 등록하고 수정할 수 있습니다.

입력할 수 있는 대표 항목:

- 제목
- 카테고리
- 태그
- Notion 링크
- 공개 범위 (`Public` / `Private`)
- 본문 내용

문서를 저장하거나 수정하면:

- 문서가 DB에 저장됩니다.
- AI 검색용 임베딩이 함께 생성될 수 있습니다.

## 9. Private 문서를 Public으로 바꾸기

본인이 만든 `Private` 문서는 필요할 때 `Public`으로 전환할 수 있습니다.

이 기능은 이런 경우에 유용합니다.

- 개인 메모로 정리해두었던 문서를 팀과 공유하고 싶을 때
- 나중에 팀 공용 문서로 전환하고 싶을 때

반대로 `Public` 문서를 다시 `Private`으로 돌릴 수도 있습니다.

## 10. 문서 검색

문서는 일반 검색과 AI 검색처럼 사용할 수 있습니다.

검색 예시:

- `ArgoCD 배포 오류`
- `QA 상태값 규칙`
- `8080 포트 충돌`

검색 시 사용할 수 있는 기준:

- 검색어
- 카테고리
- 공개 범위

AI 검색을 사용하면:

- 관련 문서 후보를 찾아주고
- 필요하면 요약 답변도 함께 보여줍니다

## 11. 문서 상세 보기

문서 상세 팝업에서는 아래 정보를 볼 수 있습니다.

- 제목
- 카테고리
- 태그
- 본문
- Notion 링크
- 공개 범위
- 작성자 정보
- 수정일

필요하면 여기서:

- 문서 수정
- 문서 삭제
- 공개 범위 변경

도 할 수 있습니다.

## 12. 계정 아카이브란?

계정 아카이브는 일반 문서 영역과 별도로 분리된 검색 전용 공간입니다.

예를 들어 이런 정보를 넣을 수 있습니다.

- 시스템 로그인 ID
- 운영 URL
- IP 주소
- 비밀번호 관련 메모
- 서비스별 접속 정보

이 공간은 민감한 정보가 들어갈 수 있어서,  
일반 서비스 승인과 별도로 한 번 더 관리자 승인을 받아야 합니다.

## 13. 계정 아카이브 접근 방법

계정 아카이브를 보려면 아래 두 가지가 모두 필요합니다.

1. 일반 서비스 승인 완료
2. 아카이브 접근 승인 완료

아직 아카이브 승인이 없으면:

- 아카이브 접근 요청 버튼이 보입니다.
- 요청 후 관리자가 승인하면 사용할 수 있습니다.

## 14. 계정 아카이브 검색

아카이브에서는 짧은 정보도 빠르게 찾을 수 있도록 AI 검색을 사용할 수 있습니다.

검색 예시:

- `es 로그인 아이디`
- `운영 DB URL`
- `Jenkins IP 정보`

AI 검색 결과에는 다음이 포함될 수 있습니다.

- 관련 항목 목록
- 짧은 요약 답변
- URL, ID, IP 같은 핵심 정보

## 15. 자주 보이는 안내 문구

### `이메일 인증 전입니다. 메일함에서 인증을 완료해 주세요.`

의미:

- 회원가입은 되었지만 이메일 인증이 아직 끝나지 않은 상태입니다.

해야 할 일:

- 받은편지함 확인
- 스팸함 확인
- 인증 링크 클릭
- 다시 로그인 시도

### `관리자 승인이 필요합니다.`

의미:

- 이메일 인증은 끝났지만 아직 팀 승인 전입니다.

해야 할 일:

- 관리자 승인 완료까지 기다리기
- 필요하면 상태 새로고침

### `이메일/비밀번호가 일치하지 않습니다.`

의미:

- 입력한 로그인 정보가 맞지 않습니다.

해야 할 일:

- 이메일 오타 확인
- 비밀번호 다시 확인

### `email rate limit exceeded`

의미:

- 인증 메일을 너무 짧은 시간 안에 여러 번 요청해서 일시 제한된 상태입니다.

해야 할 일:

- 잠시 기다린 뒤 다시 시도
- 이미 도착한 인증 메일이 있는지 받은편지함/스팸함 확인
- 버튼을 여러 번 연속으로 누르지 않기

## 16. 인증 메일 재전송

로그인 화면에는 `인증 메일 재전송` 버튼이 있습니다.

동작 방식:

- 마지막으로 입력했거나 가입에 사용한 이메일 기준으로 재전송
- 버튼 연타 방지 적용
- 재전송 후 쿨타임 적용

즉, 메일이 안 왔다고 느껴질 때 무한으로 누르는 것이 아니라,  
한 번 요청한 뒤 잠시 기다리고 메일함을 먼저 확인하는 방식으로 사용하면 됩니다.

## 17. 다시 로그인 시도 버튼

`다시 로그인 시도` 버튼은 다음 용도입니다.

- 이메일 인증을 막 끝낸 뒤 다시 로그인 확인
- 승인 상태가 바뀌었는지 다시 확인
- 같은 이메일/비밀번호로 로그인 요청 재시도

이 버튼은 현재 입력한 값으로 로그인 요청을 다시 보내는 기능입니다.  
브라우저 세션을 강제로 초기화하는 버튼은 아닙니다.

## 18. 이런 분께 맞는 서비스입니다

- 팀 문서가 Notion, 메모, 운영 기록으로 흩어져 있는 경우
- 팀원만 접근 가능한 내부 검색 사이트가 필요한 경우
- 공용 문서와 개인 문서를 한곳에서 관리하고 싶은 경우
- 계정/URL/IP 같은 짧은 운영 정보를 검색형으로 정리하고 싶은 경우

## 19. 문제가 생기면 먼저 확인할 것

1. 이메일 인증이 끝났는지 확인
2. 관리자 승인이 끝났는지 확인
3. 로그인 정보가 맞는지 확인
4. 메일 재전송을 너무 여러 번 누르지 않았는지 확인
5. 스팸함까지 확인했는지 확인

## 20. 한 줄 요약

이 서비스는:

- 팀 승인 기반 내부 문서 검색 서비스이고
- `Public`은 팀 공용 문서
- `Private`은 개인 문서
- `계정 아카이브`는 추가 승인 기반 민감 정보 검색 공간

으로 이해하시면 됩니다.
